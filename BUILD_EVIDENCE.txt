Reflection (3-5 sentences)

The Build-Verify-Commit cycle caught two real problems early. First, in
Chunk 2 the repository unit tests failed because three create() calls
landed inside the same ISO millisecond, so the sort order was decided by
UUID tiebreaker — adding a monotonic insertion counter fixed it before any
list endpoint could become flaky in production. Second, in Chunk 3 the
tsc --noEmit step rejected my first Request augmentation (the
`import "express-serve-static-core"` form), which had silently broken the
overloads of express.json(); the typecheck step caught it before any
runtime would have. I was most tempted to skip verification on the
mechanical endpoints (GET-by-id and DELETE), but writing the second
"edge case" test anyway took thirty seconds and produced the double-delete
test that proves deletion is not silently idempotent — exactly the kind of
detail that rots into a bug if no test pins it down.
